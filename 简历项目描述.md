# 门店预约与客户库存管理系统 - 简历项目描述

## 项目名称

门店预约与客户库存管理系统

## 项目简介

该项目是一个面向线下门店的本地化管理系统，主要用于客户档案管理、预约管理、服务项目管理、订单收款与欠款记录、库存管理和经营数据统计。系统支持管理员登录、账号权限拦截、只读账号访问、本地数据存储以及免安装便携式交付，适用于门店在本地电脑上独立运行和管理日常业务数据。

## 技术栈

- 后端：Spring Boot、Spring MVC、MyBatis、PageHelper、JWT
- 数据库：MySQL、H2 本地文件数据库
- 前端：Vue 3、TypeScript、Vite、Element Plus
- 数据可视化：Apache ECharts
- 构建与交付：Maven、npm、jlink、Windows BAT 脚本

## 核心功能

- 登录认证：基于 JWT 实现后台登录认证和接口访问拦截。
- 权限控制：支持管理员账号和只读账号，管理员可新增、修改、删除数据，只读账号只能查看数据。
- 客户档案：维护客户姓名、手机号、性别、生日、客户等级、来源和备注信息，支持普通客户、VIP、大客户等分类。
- 预约管理：支持客户预约服务项目，记录预约时间、服务项目数量、服务明细、预约状态和备注。
- 服务项目管理：支持服务项目新增、编辑、上下架、价格和服务时长维护。
- 订单管理：支持微信、支付宝、现金等支付方式，记录次卡、优惠金额、已付金额、欠款金额和月付金额。
- 欠款管理：支持客户购买高额项目时分期付款，例如订单金额 9800 元，每月支付 2000 元，并记录剩余欠款。
- 员工管理：支持员工信息维护、账号启用和禁用。
- 数据统计：使用 Apache ECharts 展示客户数量、预约数量、订单数量和营业金额趋势。
- 库存管理：支持库存物品新增、编辑、删除、分类、单位、当前库存、安全库存、成本价、供应商和低库存提示。
- 本地交付：支持生成免安装便携版，内置 Java 运行时和本地文件数据库，双击脚本即可启动系统。

- 设计客户档案、预约表、订单表、库存表等业务数据结构，并移除物理外键，降低交付环境下的数据迁移和维护风险。
- 实现 JWT 登录认证拦截器，对后台接口进行统一登录校验。
- 实现账号级只读权限拦截，保证只读账号无法通过接口新增、修改或删除数据。
- 完成订单支付和欠款逻辑设计，支持支付方式、优惠金额、已付金额、欠款金额、月付金额等字段维护。
- 使用 Vue 3、TypeScript 和 Element Plus 开发后台管理页面，完成客户档案、库存管理、订单管理、预约管理等页面。
- 使用 Apache ECharts 实现经营数据统计图表展示。
- 优化前端 UI 风格，提高后台系统的视觉质感和操作体验。
- 制作本地免安装交付包，集成后端 jar、前端静态资源、H2 本地数据库和 Java 运行时，实现双击启动。

## 项目亮点

- 实现了完整的门店业务闭环，覆盖客户建档、预约、服务项目、订单收款、欠款记录、库存管理和数据统计。
- 针对实际门店业务场景，支持 VIP 大客户、老客来源、抖音/美团获客、次卡消费、朋友转用、分期付款和欠款记录。
- 通过 JWT 拦截器和只读账号拦截器，实现登录认证和账号权限隔离。
- 将系统改造成便携式本地应用，用户无需安装 Java、MySQL 或 Node.js，解压后双击即可运行。
- 数据使用本地文件数据库存储，适合小型门店在单机环境下长期使用。
- 删除数据库物理外键，避免交付到不同本地环境时因外键约束导致导入失败或误操作风险。
- 前端采用 Element Plus 组件库并进行样式优化，界面更加圆滑、统一，提升了后台管理系统的专业度。

## 简历精简版

**门店预约与客户库存管理系统**

基于 Spring Boot + MyBatis + Vue 3 + Element Plus 开发的线下门店本地管理系统，包含客户档案、预约管理、服务项目、订单收款、欠款分期、数据统计和库存管理等功能。后端使用 JWT 实现登录认证和接口拦截，支持管理员和只读账号两种权限；订单模块支持微信、支付宝、现金支付、优惠金额、已付金额、欠款金额和月付金额记录；前端使用 Apache ECharts 展示经营数据趋势。项目最终制作成免安装便携版，内置 Java 运行时和 H2 本地文件数据库，用户解压后双击即可在本地运行。

## 面试可讲点

- 为什么要移除数据库物理外键：降低本地交付、数据导入和后续维护风险，改由业务逻辑控制关联关系。
- 为什么增加只读账号：满足演示和交付场景，对方可以查看数据，但不能通过系统修改数据。
- 为什么使用 H2 文件数据库做便携版：避免客户电脑安装 MySQL，实现真正的本地免安装运行。
- 为什么前端打进后端 jar：减少部署组件，用户只需要启动一个程序即可访问系统。
- 库存模块如何设计：使用库存名称、分类、单位、当前库存、安全库存、成本价、供应商和状态字段，并在前端展示低库存提示。
- 欠款逻辑如何设计：订单记录原价、优惠、应付、已付、欠款和月付金额，根据已付金额动态判断是否存在欠款。
